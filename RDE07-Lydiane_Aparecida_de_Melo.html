Branches

São ramificações de um projeto de software ou documento e tem como objetivo o desenvolvimento paralelo de:

  • Novas funcionalidades;
  • Correção de problemas;
  • Alteração de código.

Branch Master ou main: Braço principal de um projeto em um repositório.

Branch Secundária: Braço secundário de um projeto em um repositório.

Primeiramente, começamos a desenvolver o projeto, cria uma versão e faz um commit, salvando a criação da versão 1, isso está no Braço principal. Porém ai realizar uma alteração na brache master ocorrá erro, visto que, no mesmo deram usuários acessando naquele momento, acarretando em problema. Por isso, é necessário ter uma versão estável na Branch Master, enquanto a equipe do software realiza alterações através do repositório da Branch Secundária, que não irá acarretar em alterações direta na da Branch Master. Por fim, depois de concluirem as alterações é necessário realizar o merge, ou seja, a mesclagemcom a Branch Master e seguirem em fase de produção sem que ocorra qualquer tipo de problema.

Hospedagem

É de extrema importância realizar a hospedagem, porque ela consolida e digamos finaliza o trabalho desenvolvido, é como se fosse colocado o seu site em algum servidor e ele colocará em produção, ou seja, o usuário que tenha acesso a internet poderá acessar o seu site.

Para hospedar deve-se pagar pelo serviço, entre tanto o GitHub Pags oferece hospedagem gratuita de sites. Basta criar uma conta e um repositório público e terá a URL.

Nesse caso é necessario acessar a conta no GitHub no navegador, selecionar um repositório e a opção de setting.

Ao entrar na página do GitHub Pages, selecione o Branch Main e save.
Após realizar esses passos, será disponibilizado um link, onde abrirá o seu repositório na página do nagevador em hipertexto.
